  <action>
Update the `startCommand` in `railway.json` from:
```
"startCommand": "pnpm --filter server start:prod"
```
to:
```
"startCommand": "pnpm --filter server migration:latest && pnpm --filter server start:prod"
```

This ensures database migrations run before the server starts accepting traffic on every deploy. The `migration:latest` command runs Kysely migrations. If no pending migrations exist, it completes instantly (no-op).

Do NOT change any other field in railway.json. The healthcheckPath, healthcheckTimeout, builder, dockerfilePath, restartPolicyType, restartPolicyMaxRetries, and routes must remain exactly as they are.
