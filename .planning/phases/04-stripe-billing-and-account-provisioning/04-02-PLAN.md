    **user-provisioning.service.ts:**

    Injectable service with dependencies: `@InjectKysely() db: KyselyDB`, `MailService`, `UserTokenRepo` (from auth module), `EnvironmentService`.

    Interface for provision input:
    ```typescript
    interface ProvisionInput {
      email: string;
      name: string;
      spaceId: string;
      workspaceId: string;
      gateway: 'stripe' | 'kiwify';
      gatewayCustomerId?: string;    // stripe_customer_id
      gatewaySubscriptionId?: string; // stripe or kiwify subscription ID
    }
    ```

    **provision(input: ProvisionInput):**
    1. Normalize email: `input.email.toLowerCase().trim()`
    2. Check if user exists: `SELECT id, billing_locked_at FROM users WHERE email = ? AND workspace_id = ?`
    3. **If user exists (re-activation):**
       - Clear billingLockedAt: `UPDATE users SET billing_locked_at = null WHERE id = ?`
       - Check space membership: query `space_members` for userId + spaceId
       - If not a member: `INSERT INTO space_members (user_id, space_id, role) VALUES (?, ?, 'reader')`
       - Update/create billing record (see BillingService below)
       - Generate set-password token and send welcome email
       - Return existing user
    4. **If user does not exist (new account):**
       - Generate UUID for user id (use `gen_random_uuid()` or import `randomUUID`)
       - Direct Kysely insert into users: `{ email, name, password: null, workspaceId, role: 'member', locale: gateway === 'kiwify' ? 'pt-BR' : 'en-US' }`
       - DO NOT use `userRepo.insertUser()` (hashes null password)
       - Insert space_members: `{ userId, spaceId, role: 'reader' }`
       - Create billing record via BillingService
       - Generate set-password token (nanoIdGen(16), type: FORGOT_PASSWORD, 24h expiry)
       - Build setPasswordLink: `${APP_URL}/password-reset?token=${token}`
       - Send welcome email via mailService.sendToQueue()
       - Return new user

    **revoke(email: string, workspaceId: string):**
    1. Find user by email + workspaceId
    2. If not found, return (idempotent)
    3. Set `billing_locked_at = now()` on user
    4. Find space membership for this user across all spaces (query space_members by userId)
    5. Remove space membership if found

    **lock(email: string, workspaceId: string):**
    1. Find user by email + workspaceId
    2. Set `billing_locked_at = now()` (do NOT remove from space — just lock)

    **unlock(email: string, workspaceId: string, spaceId: string):**
    1. Find user by email + workspaceId
    2. Clear `billing_locked_at = null`
    3. Check if user has space membership in spaceId
    4. If not, re-add with READER role

    Use Kysely transactions for provision and revoke operations (prevent race conditions per Pitfall 6 in RESEARCH.md).

    **billing.service.ts:**

    Injectable service with `@InjectKysely() db: KyselyDB`.

    **createOrUpdateBillingRecord(params):**
    - Check if billing record exists for this workspaceId + gateway subscription ID
    - If exists: update status, subscription fields
    - If not: insert new billing record with userId, workspaceId, stripeCustomerId/kiwifyCustomerEmail, gateway, stripeSubscriptionId/kiwifySubscriptionId, status: 'active'

    **getSubscribers(workspaceId: string):**
    - Query billing table joined with users: SELECT users.id, users.email, users.name, users.billingLockedAt, billing.stripeCustomerId, billing.gateway, billing.status, billing.stripeSubscriptionId, billing.kiwifySubscriptionId FROM billing INNER JOIN users ON billing.userId = users.id WHERE billing.workspaceId = workspaceId
    - Return array for admin subscriber list

    **findBillingByCustomerId(stripeCustomerId: string):**
    - Find billing record and associated user for webhook event processing

    **findBillingByEmail(email: string, workspaceId: string):**
    - For Kiwify lookups (Kiwify identifies by email, not customer ID)
