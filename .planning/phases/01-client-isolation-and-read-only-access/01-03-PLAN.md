**Approach:** Extend the `getUsersPaginated` query in `user.repo.ts` to include a derived `userType` field via a subquery JOIN against `space_members`. The logic:
- If `user.role` is `'owner'` or `'admin'` → `userType = null` (they are Admin/Owner, not Staff or Client)
- If `user.role` is `'member'` AND the user has any `space_members` row with `role = 'writer'` or `role = 'admin'` → `userType = 'staff'`
- If `user.role` is `'member'` AND the user has ONLY `space_members` rows with `role = 'reader'` (or no space memberships) → `userType = 'client'`

**In `apps/server/src/database/repos/user/user.repo.ts`:**

1. In the `getUsersPaginated` method, add a subquery select that computes `userType`. Using Kysely's `sql` helper:

```typescript
import { sql } from 'kysely';
```

Add to the select chain:
```typescript
.select(
  sql<string | null>`
    CASE
      WHEN users.role IN ('owner', 'admin') THEN NULL
      WHEN EXISTS (
        SELECT 1 FROM space_members
        WHERE space_members."userId" = users.id
        AND space_members.role IN ('writer', 'admin')
      ) THEN 'staff'
      ELSE 'client'
    END
  `.as('userType')
)
```

This avoids N+1 queries — the subquery executes as part of the main pagination query.
